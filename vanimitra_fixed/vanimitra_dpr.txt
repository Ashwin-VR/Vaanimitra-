=========================================
SECTION 1 · WHAT WE ARE BUILDING
=========================================

1.1 Problem
Over 2.5 billion people cannot effectively use smartphone voice assistants:
no internet, unsupported language, inability to read/write, visual impairment, age.

1.2 Solution
Vanimitra (Voice Friend) is a fully offline Android voice assistant that:
- Wakes on keyword 'Vaani' — no button press needed
- Understands Hindi, Tamil, English, and code-switched speech
- Converts speech to 1 of 15 device actions via two-tier AI pipeline
- Responds in the same language the user spoke
- Learns user-specific vocabulary over time
- Works with airplane mode ON — zero internet dependency after setup
- Costs $0 per user per month

1.3 Target Users
- Visually impaired users: cannot see screen — voice is the only interface
- Elderly users: prefer speaking over touchscreens
- Low-literacy users: cannot read/type
- Rural users: no reliable internet — offline-first mandatory
- Hindi/Tamil speakers: no good assistant exists for their language offline

1.4 The 15 Supported Actions
Intent             | What it does                  | Required params
FLASHLIGHT_ON      | Turns on device torch         | none
FLASHLIGHT_OFF     | Turns off device torch        | none
VOLUME_UP          | Increases volume by 15%       | none
VOLUME_DOWN        | Decreases volume by 15%       | none
CALL_CONTACT       | Opens dialer with contact     | contact:string
SET_ALARM          | Sets alarm via Android intent | time:HH:MM
SEND_WHATSAPP      | Opens WhatsApp with message   | contact, message
OPEN_APP           | Launches app by name          | app:string
NAVIGATE           | Opens Google Maps             | destination:string
TOGGLE_WIFI        | Opens WiFi settings page      | state:on|off
TOGGLE_BLUETOOTH   | Opens BT settings page        | state:on|off
GO_HOME            | Goes to Android home screen   | none
GO_BACK            | Navigates back                | none
LOCK_SCREEN        | Opens security settings       | none
TAKE_SCREENSHOT    | Gracefully declines           | none
UNKNOWN            | Speaks help prompt            | none

ANDROID 13 NOTE: TOGGLE_WIFI and TOGGLE_BLUETOOTH cannot directly change
state — open the Settings page instead. TAKE_SCREENSHOT requires
MediaProjection — return ActionResult.fail() gracefully, never crash.

SECTION 3 · SYSTEM ARCHITECTURE
=========================================

Pipeline layers:
Layer 0: PorcupineWakeWordService — Audio stream 24/7 → onWakeWordDetected() — DEV 2
Layer 1: AudioCaptureService — Wake trigger → Raw audio (max 8s) — DEV 2
Layer 2: NativeSttService + LanguageDetector — Raw audio → transcript + language — DEV 2
Layer 3a: RuleEngine (fast path) — transcript → ParsedCommand or null — DEV 2 (<50ms)
Layer 3b: LlmService (slow path) — transcript + cacheCtx → ParsedCommand — DEV 2 (3–6s)
Layer 4: DialogueController — ParsedCommand → complete ParsedCommand — DEV 2
Layer 5: ActionExecutor — ParsedCommand → ActionResult — DEV 3
Layer 5b: TtsService — string + language → spoken audio — DEV 3
Layer 6: Personalisation + Analytics — ActionResult → SQLite log — DEV 2

Integration Contract (Dev 2 → Dev 3 boundary):
Both classes live in lib/models/ and are owned by the Lead.

ParsedCommand (Dev 2 produces, Dev 3 consumes):
- intent: VIntent (enum value, always valid, never null)
- params: Map<String,dynamic> (intent-specific, may be empty, never null)
- language: String ('en' | 'hi' | 'ta')
- source: String ('rule' | 'llm')
- rawTranscript: String
- timestamp: DateTime

ActionResult (Dev 3 produces, Dev 2 consumes):
- success: bool
- detail: String? (contact name / time / app — for TTS composition)
- error: String? (logging only, never spoken aloud)
Constructors: ActionResult.ok({String? detail}) and ActionResult.fail(String error)

Dialogue State Machine:
States: idle → listening → processing → clarifying → executing
- Max clarification attempts: 2
- Clarification timeout: 8 seconds
- Double miss → speak help → reset to idle
- WakeWordService.start() called after 300ms delay post-execution
- WakeWordService.stop() called BEFORE STT records

SECTION 10 · TtsService spec
=========================================

CRITICAL: System TTS only. No pre-recorded audio files for speech.
Only ONE audio file: assets/earcon/chime.mp3 (for earcon).

Install BEFORE event on demo device:
Settings → General Management → Text-to-speech → install hi-IN and ta-IN

TtsService singleton:
  final FlutterTts _tts = FlutterTts();
  final AudioPlayer _earconPlayer = AudioPlayer(); // earcon only

  Locale map:
    'en' → 'en-IN'
    'hi' → 'hi-IN'
    'ta' → 'ta-IN'

  init():
    _tts.setVolume(1.0)
    _tts.setSpeechRate(0.45) — slightly slow for clarity
    _tts.setPitch(1.0)
    _tts.awaitSpeakCompletion(true)

  speak(Map<String,String> strings, String language):
    Get text = TtsStrings.get(strings, language)
    Call _speakText(text, language)

  speakDynamic(String text, String language):
    Call _speakText(text, language)

  speakComposed(Map<String,String> strings, String suffix, String language):
    base = TtsStrings.get(strings, language)
    _speakText('$base $suffix', language)

  speakEarcon():
    _earconPlayer.play(AssetSource('earcon/chime.mp3'))
    Fire and forget — do NOT await

  _speakText(String text, String language) — AWAITS FULLY:
    Use Completer<void>:
      final c = Completer<void>();
      _tts.setCompletionHandler(() { if (!c.isCompleted) c.complete(); });
      await _tts.setLanguage(_localeMap[language] ?? 'en-IN');
      await _tts.speak(text);
      await c.future;

SECTION 11 · ActionExecutor spec
=========================================

Singleton. TtsService is injected via init(TtsService tts).
Receives ParsedCommand, returns ActionResult.

CRITICAL sequence for every intent:
  1. BEFORE TTS (speakEarcon or speak/speakComposed)
  2. Execute Android action
  3. OK TTS or FAIL TTS
  4. Return ActionResult

All 15 intents:

FLASHLIGHT_ON:
  before: speakEarcon()
  try: TorchLight.enableTorch()
  ok: speak(TtsStrings.flashOn, lang) → ok(detail:'flashlight')
  fail: speak(TtsStrings.generalError, lang) → fail(e.toString())
  CATCH PlatformException — Unisoc T606 may throw

FLASHLIGHT_OFF: same with disableTorch + flashOff

VOLUME_UP:
  before: speakEarcon()
  try: get volume → clamp(v+0.15, 0, 1) → setVolume(v, showSystemUI:false)
  ok: speak(volUp) → ok(detail: percent string)
  fail: speak(generalError) → fail('volume_error')

VOLUME_DOWN: same with v-0.15 + volDown

CALL_CONTACT:
  contact = params['contact'] ?? ''
  if empty: speak(contactNotFound) → fail('no_contact')
  before: speakComposed(calling, contact, lang)
  check CALL_PHONE permission — request if needed
  if denied: speak(generalError) → fail('permission_denied')
  try: launchUrl(Uri.parse('tel:$contact'))
  ok: ok(detail: contact)
  fail: speak(contactNotFound) → fail('call_failed')
  NOTE: opens dialer, does NOT auto-dial (intentional, safety)

SET_ALARM:
  time = params['time'] ?? ''
  before: speakDynamic('Alarm for $time', lang)
  parse HH:MM → hour, minute
  try: AndroidIntent(action:'android.intent.action.SET_ALARM',
       arguments: {HOUR:h, MINUTES:m, SKIP_UI:true})
  ok: speak(alarmSet) → ok(detail: time)
  fail: speak(generalError) → fail('alarm_failed')

SEND_WHATSAPP:
  contact = params['contact'] ?? '', message = params['message'] ?? ''
  if contact empty: speak(contactNotFound) → fail('no_contact')
  before: speakComposed(calling, contact, lang) [reuse calling string]
  try: clean number, wa.me URL, launchUrl(externalApplication)
  ok: speak(msgSent) → ok(detail: contact)
  fail: speak(contactNotFound) → fail('whatsapp_failed')

OPEN_APP:
  appName = params['app']?.toLowerCase() ?? ''
  before: speakComposed(opening, appName, lang)
  package = AppRegistry.resolvePackage(appName)
  if null: speak(appNotFound) → fail('unknown_app')
  try: AndroidIntent(MAIN, package, FLAG_ACTIVITY_NEW_TASK)
  fallback if fails: launchUrl(market://details?id=package)
  ok: ok(detail: appName)
  fail: speak(appNotFound) → fail('launch_failed')

NAVIGATE:
  dest = params['destination'] ?? ''
  before: speak(navigating, lang)
  try: google.navigation:q=encoded, fallback to maps search URL
  ok: ok(detail: dest)
  fail: speak(generalError) → fail('navigate_failed')

TOGGLE_WIFI:
  before: speakEarcon()
  speak(wifiSettings, lang)
  AndroidIntent(action: 'android.settings.WIFI_SETTINGS')
  ok(detail: state)

TOGGLE_BLUETOOTH:
  before: speakEarcon()
  speak(btSettings, lang)
  AndroidIntent(action: 'android.settings.BLUETOOTH_SETTINGS')
  ok(detail: state)

GO_HOME:
  before: speakEarcon() + speak(goingHome, lang)
  AndroidIntent(MAIN, category:HOME, FLAG_ACTIVITY_NEW_TASK)
  ok(detail: 'home')

GO_BACK:
  before: speakEarcon() + speak(goingBack, lang)
  SystemChannels.navigation.invokeMethod('SystemNavigator.pop')
  ok(detail: 'back')

LOCK_SCREEN:
  before: speakEarcon() + speak(locking, lang)
  AndroidIntent(action: 'android.settings.SECURITY_SETTINGS')
  ok(detail: 'lock')
  NOTE: direct lock needs DeviceAdmin — settings page is demo-safe

TAKE_SCREENSHOT:
  No before. No action.
  return ActionResult.fail('screenshot_needs_accessibility')
  NEVER CRASH

UNKNOWN:
  speak(TtsStrings.unknown, lang)
  return ActionResult.fail('unknown_intent')
